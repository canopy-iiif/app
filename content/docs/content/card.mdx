# Card

Use `<Card>` when you need a simple linked figure: image plus caption, wrapped in an anchor. The component is layout-focused and does not fetch IIIF data on its own. Pass the URL, image, and text explicitly.

## Props

| Prop        | Type     | Required | Notes                                                    |
| ----------- | -------- | -------- | -------------------------------------------------------- |
| `href`      | string   | âœ…       | Destination URL.                                        |
| `title`     | string   |          | Primary caption text (rendered in `<figcaption>`).       |
| `subtitle`  | string   |          | Optional second line of caption text.                    |
| `src`       | string   |          | Image URL (IIIF thumbnail, static JPG, etc.).            |
| `alt`       | string   |          | Accessible alt text (falls back to `title`).             |
| `imgWidth` / `imgHeight` | number |          | Provide intrinsic dimensions to maintain aspect ratio. |
| `aspectRatio` | number |          | Alternative way to lock the image ratio.                |
| `lazy`      | boolean  |          | Defaults to `true`. Set to `false` when used in carousels. |

## Example

```mdx
<Card
  href="/works/the-town-and-pass-of-boondi-in-rajpootana.html"
  src="https://api.dc.library.northwestern.edu/api/v2/works/4bdb5a22-6c7f-498d-8e6e-e49ea9bc4778/thumbnail"
  imgWidth={1024}
  imgHeight={768}
  title="The town & pass of Boondi, in Rajpootana"
  subtitle="Lieutenant-Colonel Thomas Bacon, 1836"
/>
```

`<ReferencedItems />` renders a grid of cards automatically, so you only need to use `<Card>` directly for bespoke layouts.

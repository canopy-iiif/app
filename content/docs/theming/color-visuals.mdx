# Color & visuals

The design system inherits palettes from [Radix Themes](https://www.radix-ui.com/themes/docs/theme/color).
Pick an accent and gray family in `canopy.yml â†’ theme` to set the global brand color and neutral
surfaces.

```yaml filename="canopy.yml" {12-18}
theme:
  accentColor: tomato
  grayColor: sand
  appearance: light
```

## Accent palettes

Available options: `amber`, `blue`, `bronze`, `brown`, `crimson`, `cyan`, `gold`, `grass`, `green`,
`indigo`, `lime`, `mint`, `olive`, `orange`, `pink`, `plum`, `purple`, `red`, `sage`, `sand`, `sky`,
`slate`, `teal`, `tomato`, `violet`, `yellow`.

Accent colors power buttons, links, focus rings, pills, and slider handles. Choose one with enough
contrast against your gray palette to keep UI elements distinct.

## Gray palettes

Available options: `slate`, `gray`, `mauve`, `sage`, `olive`, `sand`.

The gray palette controls surfaces, copy, outlines, and component chrome. Pair warmer grays (sand,
mauve) with muted accents, or stick with `slate`/`gray` when you want a neutral foundation.

## Light vs dark appearance

Switch `appearance` to `dark` to render all components against dark backgrounds. The builder toggles
`color-scheme` accordingly, so form controls, scrollbars, and system UI follow suit.

## Extending radius and scale

If you need different border radii or sizing ramps, extend them in CSS rather than editing the build
pipeline:

```css
@theme {
  --radius-button: 999px;
  --font-size-base: 1.05rem;
}

button {
  border-radius: var(--radius-button);
}
```

## Preview changes quickly

- Run `npm run dev` and tweak `canopy.yml` values. The dev watcher refreshes the theme without a full
  rebuild.
- Use browser dev tools to inspect the generated CSS variables. Tokens are prefixed with
  `--color-scale-` (for ramps) and `--accent-a-` (for alpha values).
